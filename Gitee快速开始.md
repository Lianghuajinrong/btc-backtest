# Gitee 快速开始指南

## 🚀 5 分钟快速部署

### 第一步：创建 Gitee 仓库（2分钟）

1. 访问 https://gitee.com 并登录
2. 点击右上角 **"+"** → **"新建仓库"**
3. 填写信息：
   - 仓库名称：`btc-backtest`
   - 可见性：**公开**
   - 其他选项：**不勾选**
4. 点击 **"创建"**
5. **复制仓库地址**（HTTPS）：`https://gitee.com/您的用户名/btc-backtest.git`

### 第二步：配置认证（1分钟）

#### 方法 A：使用个人访问令牌（推荐）

1. Gitee → 右上角头像 → **"设置"** → **"安全设置"** → **"私人令牌"**
2. 点击 **"生成新令牌"**
3. 描述：`本地开发`
4. 权限：勾选 **`projects`**
5. 点击 **"提交"**
6. **复制令牌**（只显示一次！）

### 第三步：运行部署脚本（2分钟）

1. **打开 PowerShell**（在项目目录）
2. **运行脚本**：
   ```powershell
   .\部署到Gitee.ps1
   ```
3. **按提示操作**：
   - 输入 Gitee 仓库地址
   - 输入提交信息（或直接回车）
   - 确认推送（输入 Y）
   - **用户名**：您的 Gitee 用户名
   - **密码**：使用刚才复制的**个人访问令牌**（不是账户密码）

### 第四步：配置 Gitee Pages（1分钟）

1. 在 Gitee 仓库页面，点击 **"服务"** 标签
2. 选择 **"Gitee Pages"**
3. 配置：
   - 部署分支：`main`
   - 部署目录：`/`（如果 index.html 在根目录）
   - 或 `/10000`（如果 index.html 在 10000 目录）
4. 点击 **"启动"**
5. 等待 1-3 分钟，获取访问地址

---

## ✅ 完成！

现在您可以：
- ✅ 访问前端页面：`https://您的用户名.gitee.io/btc-backtest/`
- ✅ 代码已推送到 Gitee
- ⏳ 后端 API 需要单独部署（参考完整指南）

---

## 📝 下一步：部署后端

后端需要单独部署，推荐方案：

1. **Render**（如果网络允许）：https://render.com
2. **Railway**：https://railway.app  
3. **国内云服务**：阿里云、腾讯云、华为云

部署后端后，记得更新 `index.html` 中的 API 地址！

---

## 🆘 遇到问题？

查看详细指南：`Gitee部署完整指南.md`

常见问题：
- **推送失败**：检查访问令牌是否正确
- **Pages 无法访问**：确认仓库是公开的
- **前端无法连接后端**：检查 API 地址配置
